# Product

## Register

product

## Users

个人投资者。工作日盘前或盘中，在手机或电脑上快速查看持仓基金的预测方向、市场行情和自选列表。他们的核心诉求是高效获取决策参考信息，而非深度研究或高频交易。他们可能在通勤、午休或睡前使用，注意力有限，需要界面一目了然。

典型场景：
- 通勤地铁上单手操作手机，30 秒内扫一眼自选基金涨跌和预测方向
- 午休时在电脑上搜索某只基金，查看预测详情和模型覆盖状态
- 盘后回顾当日行情排行，判断市场整体走势

## Product Purpose

为个人投资者提供基金预测和市场行情的一站式参考工具。通过机器学习模型给出涨跌方向预测，结合基线估计和模型覆盖状态，让用户对每只基金的预测可信度有清晰认知。同时提供股票行情、K线、资金流向等辅助信息，帮助用户形成更完整的判断。

成功标准是用户能快速找到目标基金、理解预测含义、判断预测可靠性，而不被误导或产生虚假信心。

## Brand Personality

冷静，专业，可靠。

- 冷静：界面克制，信息密度适中，不制造紧迫感或 FOMO 情绪。颜色和动画都服务于信息传达，不服务于情绪刺激。
- 专业：数据呈现严谨，涨跌幅来源明确标注（实时/估值/收益），模型覆盖状态诚实披露，不隐藏不确定性。
- 可靠：预测结果始终附带置信度和数据质量说明，模型未覆盖时明确告知而非模糊带过。

## Anti-references

- 传统券商终端：不要密密麻麻的数字表格、六屏同看、红绿闪烁的 Level-2 行情。信息要有呼吸感，每屏聚焦一个任务。
- 游戏化装饰：不要把严肃的金融预测做得像游戏。避免成就徽章、连胜动画、进度条奖励等机制，这些会误导用户轻视风险。
- 炫技式金融 UI：不要花哨的 3D 图表、粒子动画、全屏渐变。装饰性视觉元素会分散投资决策的注意力。
- AI 生成感：不要紫色渐变、发光阴影、过度圆角、gradient text。界面应像工程师写的工具，不像 AI 生成的模板。

## Design Principles

1. 诚实优于乐观。模型未覆盖时明确标注，低置信度预测不伪装成高置信度。宁可让用户失望于"暂无预测"，也不让用户误信一个不可靠的数字。
2. 一屏一任务。每个页面聚焦一个核心任务：搜索基金、查看预测、管理自选、浏览行情。避免信息过载和功能堆砌。
3. 数据有来源，变化有原因。涨跌幅标注行情来源和日期，预测标注模型覆盖状态，让用户判断信息时效性和可信度。
4. 克制即尊重。动画服务于状态转换而非装饰，颜色服务于语义区分而非视觉刺激。用户的时间宝贵，界面不应浪费它。

## Information Architecture

### 核心页面

| 页面 | 核心任务 | 入口 |
|------|---------|------|
| 自选 | 快速扫视持仓基金/股票的涨跌和预测方向 | dock 首页 |
| 行情 | 浏览市场指数和涨跌排行 | dock 行情 |
| 预测 | 搜索基金并查看预测详情 | dock 预测 |
| 基金详情 | 深入了解单只基金的业绩、持仓、风险 | 行情/自选点击 |
| 股票详情 | 查看股票行情、K线、资金流向、财务 | 行情/自选点击 |

### 导航模型

底部浮动 dock（macOS 风格），3 个一级入口：自选、行情、预测。详情页通过列表点击进入，无独立导航入口。搜索通过顶栏触发全屏浮层。

### 数据层级

```
市场指数（A股/港股/美股）  ← 顶层概览
  └─ 涨跌排行（基金/股票）  ← 分类浏览
      └─ 资产详情           ← 深入研究
          ├─ 基本信息与实时行情
          ├─ 预测结果与置信度
          └─ 历史数据与财务指标
```

## Content Strategy

### 数据标注规范

| 数据类型 | 标注要求 | 示例 |
|---------|---------|------|
| 实时行情 | 来源 + 更新时间 | "东方财富 · 14:32" |
| 估值净值 | "估值" 前缀 + 估值时间 | "估值 1.2345 · 14:32" |
| 预测结果 | 信号状态 + 模型覆盖 | "可行动信号 · 模型覆盖" |
| 历史收益 | 区间起止日期 | "近1年 +12.34% (2025.01-2025.12)" |
| 模型未覆盖 | 明确标注原因 | "模型暂未覆盖 · 显示基线估计" |

### 空状态处理

| 场景 | 展示内容 |
|------|---------|
| 自选为空 | 引导添加自选的空状态插图 + 操作提示 |
| 数据加载中 | 骨架屏（shimmer），保留页面结构 |
| 数据加载失败 | 错误提示 + 重试按钮，不显示空白 |
| 预测不可用 | "暂无预测" + 原因说明，不隐藏区域 |
| 分时数据不可用 | "分时数据暂不可用" 提示，不显示空白图表 |

## Interaction Patterns

### 搜索

全屏浮层，支持基金代码/名称/拼音搜索。300ms 防抖。键盘导航（上下箭头 + Enter + Escape）。搜索历史持久化。

### 自选管理

一键添加/移除自选，无需确认弹窗。操作结果通过按钮状态变化（实心/空心星标）即时反馈。

### 数据刷新

- 行情页：30 秒自动刷新，页面隐藏时暂停
- 自选页：手动刷新，5 秒节流防重复请求
- 详情页：进入时加载，支持手动重试
- 全局：右下角浮动刷新按钮

### 卡片折叠

详情页卡片支持折叠/展开，使用弹性过渡动画。默认展开，用户可按需收起不关心的区域。

## Accessibility & Inclusion

- 目标 WCAG 2.1 AA 级别。
- 涨跌方向不仅依赖颜色（红/绿），同时使用箭头图标和文字标签。
- 支持暗色和亮色主题切换，适应不同环境光。
- 图表提供数据表格替代视图（后续迭代）。
- 尊重 prefers-reduced-motion 系统设置，全局和组件级降级。
- 尊重 prefers-reduced-transparency，移除毛玻璃效果改用纯色背景。
- 搜索支持键盘导航（ArrowUp/Down/Enter/Escape）。
- 错误状态提供重试按钮，不依赖仅视觉的提示。
- 数值使用等宽字体（JetBrains Mono），便于对齐比较。

## Technical Context

### 前端技术栈

- Vue 3 + TypeScript + Vite
- Pinia 状态管理（Composition API 风格）
- Vue Router（HTML5 History 模式，懒加载）
- ECharts（按需引入，Canvas 渲染）
- Element Plus（按需引入，暗色主题覆盖）
- CSS 变量设计令牌体系（亮/暗双主题）

### 后端技术栈

- Go + Gin 框架
- 服务层 + 仓库层分离
- 泛型缓存（DetailCache[T]）
- CSRF 防护 + 速率限制 + gzip 压缩
- 熔断器 + 请求 ID 追踪

### 模型服务

- Python FastAPI 独立部署
- 通过 HTTP API 与 Go 后端通信
- 熔断器保护（3 次失败断开，30 秒半开试探）
- 降级策略：模型不可用时显示基线估计
