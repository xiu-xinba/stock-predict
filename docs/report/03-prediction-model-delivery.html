结论

工程闭环已达到可交付状态。 当前实现已经覆盖设计文件要求的收益拆解、模型锦标赛、滚动回测、分市场状态评估、经验残差预测区间、低置信/无信号控制、预测日志、监控和 shadow/challenger 评估。收益拆解和预测区间已从 Python 模型服务贯穿到 Go API 与前端预测卡片。

模型效果仍受样本跨度限制。 现有 public MVP 数据可验证链路，但不能等同于稳定实盘质量；弱模型会被晋级门槛和在线可行动闸门拦住，避免只因单次置信度偏高或极少数样本看似准确就输出 is_actionable=true。

剩余风险

  • 当前 public MVP 数据跨度仍偏短，尤其日内模型需要更多交易日覆盖不同波动和恐慌状态。
  • 深度模型栈，如 TFT、PatchTST、DeepLOB、Temporal GNN，尚未进入主训练流程；需要更大数据集后再接入。
  • ONNX sidecar 当前因本地缺少 sklearn-onnx 依赖而跳过，不影响 Python 模型服务推理。
  • 前端生产构建通过，但 ECharts vendor chunk 仍超过 Vite 默认 500 kB 提醒阈值；这不阻塞 MVP 验收，可在交付后做按路由懒加载或 chunk 拆分。
