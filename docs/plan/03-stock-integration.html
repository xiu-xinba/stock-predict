股票功能扩展 — 设计执行计划

版本 1.0  |  2026-05-29  |  基金预测项目 → 综合金融信息平台

一、项目概述

将现有基金预测项目扩展为基金 + 股票综合性金融信息平台。股票功能与基金功能在结构上保持一致性，信息深度和广度达到平衡。采用统一路由 + Tab 切换方案，复用现有 UI 框架和设计语言。

1.1 设计决策

决策项选择理由
数据源东方财富 + 腾讯行情与基金数据源一致，免费且稳定
页面结构统一路由 + Tab 切换用户体验一致，开发成本可控
详情页6 卡片对称与基金详情结构对称，信息完整
预测功能独立股票模型股票与基金特征不同，需专用模型

六、股票 vs 基金 — 结构对照

维度基金股票
代码格式6位数字（110011）6位数字（600519）
市场标识无（基金统一市场）sh/sz/bj
行情来源腾讯行情 + 天天基金估值腾讯行情（直接）
详情页路由/fund/:fundCode/stock/:stockCode
详情卡片数6（头部+业绩+经理+组合+风险+预测）6（头部+行情+K线+资金+财务+预测）
预测端点/predict/:fundCode/stock/:stockCode/predict
自选存储localStorage fund_watchlistlocalStorage stock_watchlist
排行类型gainers/losersgainers/losers/volume

七、风险与缓解

风险影响缓解措施
东方财富 API 限流股票数据获取失败5分钟缓存 + 请求间隔控制 + 降级为基线数据
股票代码与基金代码冲突搜索结果混淆通过市场前缀区分（sh600519 vs 110011）
K线数据量大前端渲染性能默认加载日K 250条，懒加载更多
股票模型未训练预测功能不可用先使用基线预测，模型就绪后切换
