概述

在现有项目基础上，构建全面的基金数据库与搜索系统。核心目标：扩展数据库模型、实现数据同步、增强搜索能力、优化前端体验。

架构

数据流

东方财富API → FundSyncService → SQLite数据库 → FundService(搜索/筛选/排序) → FundsController → 前端
                                                                    ↑
                                                              IMemoryCache

同步端点：POST /api/v1/funds/sync（管理员）

触发手动同步基金数据。

筛选选项端点：GET /api/v1/funds/filters

返回可用的筛选选项（基金类型列表、公司列表、风险等级列表）。

数据同步

FundSyncService

  • 启动时：检查数据库基金数量，若 < 100 则触发全量同步
  • 定时任务：每24小时增量同步（更新净值和收益率）
  • 数据源：东方财富基金列表API + 天天基金详情API

同步策略

  1. 全量同步：从东方财富基金列表API分页获取所有基金基本信息
  2. 增量更新：仅更新净值和收益率等时效性数据
  3. 错误处理：单条基金同步失败不影响整体，记录日志

前端设计

搜索组件增强（嵌入MarketView）

  1. 搜索栏：保留现有输入框，增加筛选图标按钮
  2. 筛选面板：点击筛选图标展开，包含类型、公司、风险等级下拉选择
  3. 搜索建议：下拉列表增加净值、涨跌幅、风险等级信息
  4. 搜索历史：localStorage 存储最近10条搜索记录，显示在建议列表顶部
  5. 排序切换：搜索结果支持按收益率、净值等排序

性能保障

  1. 数据库索引：FundName、FundType、Company、RiskLevel 建立索引
  2. 搜索缓存：IMemoryCache 缓存搜索结果，TTL 60秒
  3. 筛选选项缓存：IMemoryCache 缓存筛选选项，TTL 300秒
  4. 分页查询：所有搜索结果分页返回，避免大量数据传输
  5. API响应时间目标：< 500ms（P95）
