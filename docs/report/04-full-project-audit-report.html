七、架构级风险

训练-推理一致性风险 (Training-Serving Skew)
训练用 joblib/pickle，推理也用 joblib，但 ONNX 导出路径存在却未在推理中使用。如果未来切换到 ONNX 推理，特征预处理逻辑可能不一致，导致预测结果偏差。
全链路测试缺失
Python 模型训练模块虽然有 test 文件，但核心的数据泄漏防护、特征一致性、标签构建正确性等关键路径缺乏验证。Go 后端 market_service、model_client 等核心模块无测试。任何重构都无法保证正确性。
单点故障与无容错
模型服务为单实例，无健康检查、无重试、无熔断；Go 后端无优雅关闭（graceful shutdown），进程中断时正在进行的写操作可能损坏数据。
模型服务为单实例，无健康检查、无重试、无熔断；Go 后端无优雅关闭（graceful shutdown），进程中断时正在进行的写操作可能损坏数据。
配置散落无统一管理
Go 后端环境变量、Python YAML 配置、前端 Vite 环境变量三套配置体系无统一管理。模型服务 URL、超时时间、CORS 源等关键配置在不同模块中独立定义，修改时容易遗漏。
