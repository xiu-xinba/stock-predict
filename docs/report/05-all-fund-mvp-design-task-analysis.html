设计文档摘要

全基金 MVP 数据覆盖与训练准备设计

背景：当前系统已有约 2.68 万只基金的本地搜索库，其中约 1.96 万只有东方财富排行口径的最新净值、涨跌幅和阶段收益。预测模型层仍以指数 ETF 为主。

三层架构

  • 全库可搜索：所有基金都能被搜索、选择、进入预测入口
  • 全库可解释：每只基金都返回明确的数据质量和模型覆盖状态
  • 分层可训练：只有具备足够历史净值和合适标签的基金进入日频 MVP 模型

实施顺序（7 步）

  1. 增加全基金覆盖报告和验收脚本
  2. 增加通用基金历史净值采集器
  3. 增加 all_fund_nav_daily_v1 样本构建
  4. 训练一个通用日频 MVP baseline/champion
  5. 后端按覆盖清单路由模型或基线
  6. 前端补齐覆盖状态和行情日期展示
  7. 重启服务，执行搜索、预测、构建、测试和健康检查
